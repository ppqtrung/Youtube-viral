# YouTube Viral Advisor

A data-driven tool that helps YouTube creators estimate — *before hitting publish* — the probability that their next video lands in the **top 25 % of trending videos**. The project combines a cleaned multi-country trending dataset, a calibrated gradient-boosted classifier, NLP-derived title features, and a polished Streamlit front-end for exploration and prediction.

## Overview

Most creators rely on intuition when choosing a title, picking tags, or scheduling an upload. This project replaces that guesswork with an **explainable, calibrated virality score** and a set of concrete, data-driven recommendations.

The product has three pages, all served through a single Streamlit app:

1. **Viral Advisor** — the core product. Enter a title, category, country, tag count, and publish time; get a probability score, a benchmark table, targeted recommendations, and detected viral keywords.
2. **Dashboard** — analytics across the entire trending dataset: category performance, timing patterns, engagement distributions, and viral vs non-viral comparisons.
3. **Model Insights** — transparency view showing ROC/PR AUC, the calibration curve, permutation feature importance, per-class classification metrics, and the top-lift viral keywords.

## Dataset

- **Source:** YouTube trending snapshots across **11 countries** — `BR`, `CA`, `DE`, `FR`, `GB`, `IN`, `JP`, `KR`, `MX`, `RU`, `US`.
- **Raw files:** [raw_data/](raw_data/) contains one `*_Trending.csv` per country (≈ 228 MB total) plus a [data_dictionary.csv](raw_data/data_dictionary.csv) describing the columns.
- **Processed output:** [data/processed.csv](data/processed.csv) — cleaned, feature-engineered, and ready for both the model and the app. Generated by `analysis.py`.
- **Scale after cleaning:** ~178 K videos across 15 categories (the dashboard's "Dataset snapshot" section displays the exact counts from the current build).

Raw columns of interest: `video_id, title, channel_title, views, likes, comments, publish_time, category_id, tags`.

## How it works

### 1. Data pipeline — [rq2_project/analysis.py](rq2_project/analysis.py)

Reads every `*_Trending.csv` from [raw_data/](raw_data/), tags each row with a `country` code derived from the filename, and then:

- Maps `category_id` → human-readable `category` using a built-in YouTube category map (`YOUTUBE_CATEGORY_MAP`).
- Parses `publish_time` into UTC and derives `publish_hour` + `publish_dow` (0 = Monday).
- Coerces `views`, `likes`, `comments` to `int64` and drops rows with `views == 0`.
- Splits the `tags` cell (`|`, `,`, or `;` delimited) into a list and stores `number_of_tags`.
- Computes **title NLP features** in a single pass via `_title_features`:
  - `title_length` — character count
  - `title_word_count` — word count
  - `title_caps_ratio` — fraction of ALL-CAPS words (len ≥ 2)
  - `title_has_number` — 0/1 flag for any digit
  - `title_has_question` — 0/1 flag for `?`
  - `title_has_exclamation` — 0/1 flag for `!`
- Derives engagement ratios: `like_ratio = likes / views`, `comment_ratio = comments / views`, both clipped to `[0, 1]`.
- Writes the final dataframe to [data/processed.csv](data/processed.csv).

### 2. Model training — [rq2_project/model.py](rq2_project/model.py)

**Target.** Binary `is_viral` = 1 when `views ≥ 75th percentile` of the training set (≈ 25 % positive class).

**Features used (all pre-publish).**

| Kind | Features |
|---|---|
| Numeric (9) | `title_length`, `title_word_count`, `title_caps_ratio`, `title_has_number`, `title_has_question`, `title_has_exclamation`, `number_of_tags`, `publish_hour`, `publish_dow` |
| Categorical (2) | `category`, `country` (one-hot encoded) |

**Pipeline.**

1. `ColumnTransformer` — one-hot encodes categoricals, passes numerics through.
2. `HistGradientBoostingClassifier` — 300 iterations, `max_depth=8`, `learning_rate=0.08`, `min_samples_leaf=30`, `l2_regularization=1.0`, early stopping on a 10 % validation split.
3. `CalibratedClassifierCV` — isotonic calibration with 3 inner folds, so predicted probabilities are meaningful (directly interpretable as "chance of viral").

**Evaluation and artefacts.**

- 5-fold stratified cross-validated ROC AUC on the full dataset.
- 80/20 stratified train/test split, reporting ROC AUC, PR AUC, Brier score, and a full classification report.
- **Permutation feature importance** on a 3 000-row test subsample (10 repeats, `scoring="roc_auc"`).
- **Calibration curve** (10 quantile bins).
- **Viral keywords** — `extract_viral_keywords` computes a Laplace-smoothed lift `P(word | viral) / P(word)` for every word ≥ 50 occurrences, filtered through an English stop-word list.

All of the above are persisted to:

- [models/viral_model.joblib](models/viral_model.joblib) — the fitted `Pipeline`.
- [models/model_metrics.json](models/model_metrics.json) — metrics, importances, calibration points.
- [models/viral_keywords.json](models/viral_keywords.json) — top 40 lift-ranked words for the NLP panel.

### 3. Streamlit app — [rq2_project/app.py](rq2_project/app.py)

A single-file Streamlit app styled as a SaaS product (Inter font, slate palette, single accent blue, custom Altair theme). Three pages selected from the sidebar radio:

- **Viral Advisor.** Renders the model-quality KPI cards, collects user inputs via `st.form`, builds a single-row `DataFrame` matching the training schema, calls `model.predict_proba`, and displays:
  - A **score module** with a calibrated probability and an assessment band (`High / Moderate / Below average`).
  - A **benchmark table** comparing each input against the dataset's viral ranges (25th–75th percentile among videos in the top quartile of views).
  - A **recommendations list** generated by `build_suggestions` — only fires tips when an input falls outside the viral benchmark.
  - A **viral-keywords panel** showing which words in the user's title appear in the lift-ranked keyword list (highlighted chips) plus unused high-lift words (suggestion chips).
  - A **trending tags chart** — most common tags among viral videos, globally or filtered to the chosen category.
  - A **publish-hour chart** with a red rule marking the user's chosen hour against the average-views curve.

- **Dashboard.** KPI strip (video count, categories, median like/comment ratio), top-10 category breakdowns, timing charts (by hour, by day of week), and viral-vs-non-viral distribution stacks for `number_of_tags` and `title_length`.

- **Model Insights.** Metric cards, an architectural summary, side-by-side permutation-importance bar chart and calibration curve, a per-class precision/recall/F1 table, and a lift-ranked viral-keywords chart.

Loaders use `@st.cache_data` and `@st.cache_resource` so the app stays responsive after the first load.

## Quick start

Prerequisite: **Python 3.10 or newer** on `PATH`.

### Option A — one-shot installer (Windows)

From the project root [d:/hackthon1/](./), run either:

```powershell
# PowerShell
.\setup_dependencies.ps1
```

```bat
REM cmd.exe
setup_dependencies.bat
```

Both scripts create `.venv`, upgrade pip, and install everything from [rq2_project/requirements.txt](rq2_project/requirements.txt).

### Option B — manual setup

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install --upgrade pip
pip install -r rq2_project/requirements.txt
```

On macOS / Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r rq2_project/requirements.txt
```

## Running each component

Run these from the project root (`d:/hackthon1`) with the virtual environment activated.

### 1. Build the processed dataset

```powershell
python rq2_project/analysis.py
```

Reads [raw_data/](raw_data/), writes [data/processed.csv](data/processed.csv). Prints row counts before and after cleaning and the number of distinct categories.

### 2. Train the model

```powershell
python rq2_project/model.py
```

Produces [models/viral_model.joblib](models/viral_model.joblib), [models/model_metrics.json](models/model_metrics.json), and [models/viral_keywords.json](models/viral_keywords.json). Expect a few minutes of runtime on a typical laptop.

### 3. Launch the Streamlit app

```powershell
streamlit run rq2_project/app.py
```

Opens at `http://localhost:8501`. If `processed.csv` or `viral_model.joblib` is missing, the app prints a clear message telling you which script to run next.

## Troubleshooting

**`Missing data/processed.csv` error on app launch.** Run `python rq2_project/analysis.py` first.

**`Missing models/viral_model.joblib` error on app launch.** Run `python rq2_project/model.py` after the processed data exists.

**`pip install` fails on Windows with a permissions error.** Re-run the setup script from an elevated PowerShell, or activate the `.venv` manually and install from there.

**`streamlit: command not found` after install.** The environment isn't activated. Activate it with `.\.venv\Scripts\Activate.ps1` (PowerShell) or `.\.venv\Scripts\activate.bat` (cmd.exe), or invoke directly: `.\.venv\Scripts\python -m streamlit run rq2_project/app.py`.

**Permutation importance step is slow.** Expected — it runs 5 repeats on a 3 000-row subsample, typically 30 – 90 seconds.

**Port 8501 already in use.** Launch on a different port: `streamlit run rq2_project/app.py --server.port 8502`.
